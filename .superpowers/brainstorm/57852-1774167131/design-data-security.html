Tabelle SQLite

bookings
Campo Tipo Note

Campo Tipo Esempio
time_slot TEXT PK "12:00", "12:30" ... "21:00"

is_active BOOLEAN per chiusure straordinarie

Sicurezza — layer per layer

Input validation (Pydantic)
Ogni campo validato: email formato RFC, date non nel passato, party_size 1–10, name max 100 char. Rifiuto immediato se non valido.
Rate limiting (slowapi)
Max 5 richieste/minuto per IP su POST /bookings. Previene spam e bot che occupano tutti gli slot.
CORS ristretto
Backend accetta solo richieste dal dominio del frontend (Vercel URL). Nessun accesso da origini sconosciute.
Conferma via email (token UUID)
Prenotazione resta "pending" finché l'utente non clicca il link. Token scade dopo 24h. Elimina prenotazioni false.
Nessun dato superfluo
Solo i dati strettamente necessari. No password, no pagamenti, no tracking. GDPR-friendly by design.

Logica slot disponibili

GET /slots?date=2026-03-25

→ Prende tutti gli slot attivi dalla config
→ Per ogni slot: conta le booking confirmed per quella data
→ Se count < max_bookings → slot disponibile (mostrato)
→ Se count = max_bookings → slot pieno (non mostrato)
→ Se nessuno slot disponibile → messaggio "Nessuna disponibilità per questa data"
